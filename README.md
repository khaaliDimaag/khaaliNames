# khaaliNames
> An onchain library for generating unique, human-readable names

`khaaliNames` is an onchain name generator developed for [khaaliDimaag
projects](https://github.com/khaaliDimaag) to generate random usernames by combining words from 
onchain dictionaries. inspired by the 
[unique-names-generator](https://www.npmjs.com/package/unique-names-generator) npm package.

> [!TIP]
> Try out the functions at [`0xa0715EC44766D28ec2Bc8c7e94716A202937d4F2`](https://sepolia.etherscan.io/address/0xa0715ec44766d28ec2bc8c7e94716a202937d4f2#readContract)

## How it works

Names are generated by combining words from three dictionaries (colors, adjectives, animals) with 
a numeric suffix ~ _blue-bold-fox-3_


### NameTypes

The contract supports different **name types** with varying uniqueness:

| NameType | Format | Unique combinations |
| :------: | :----: | :-----------------: |
| `ANIMAL` | `fox-n` | 350 |
| `COLOR` | `blue-n` | 50 |
| `ADJECTIVE` | `bold-n` | 1,200 |
| `COLOR_ANIMAL` | `blue-fox-n` | 17,500 |
| `ADJECTIVE_ANIMAL` | `bold-fox-n` | 420,000 |
| `COLOR_ADJECTIVE_ANIMAL` | `blue-bold-fox-n` | 21,000,000 |
| `ADJECTIVE_COLOR_ANIMAL` | `bold-blue-fox-n` | 21,000,000 |

### Milestones

For convenience, **Milestone** presets map to recommended `(NameType, n)` pairs that scale with 
project growth:

| Milestone | Format | Total unique names | Significance |
| :-------: | :----: | :----------------: | :----------- |
| `ANIMAL_30` | `animal-n` (n in 1-30) | ~10,500 | first ~10k users |
| `COLOR_ANIMAL_5` | `color-animal-n` (n in 1-5) | ~87,500 | first ~100k users |
| `ADJ_ANIMAL_2` | `adj-animal-n` (n in 1-2) | ~840,000 | first ~1M users |
| `COLOR_ADJ_ANIMAL_3` | `color-adj-animal-n` (n in 1-3) | ~63,000,000 | first ~50M users |

..and so on.

This can also be useful to quickly identify which users were early supporters and reward them 
accordingly.


## Usage

> [!IMPORTANT]
> `n` is defined as a `uint8` and in practice if you have 256 collisions, you should probably pick 
> a new variation of the name

```solidity
import {IkhaaliNamesV1, Milestone, NameType} from "khaaliNames/IkhaaliNamesV1.sol";

IkhaaliNamesV1 names = IkhaaliNamesV1(DEPLOYED_ADDRESS);

// Using a milestone preset (recommended)
string memory name = names.getRandomName(userAddress, Milestone.ANIMAL_30);

// Using the escape hatch (custom NameType + suffix range)
string memory name = names.getRandomName(userAddress, NameType.COLOR_ANIMAL, 5);
```

## Architecture
> Note the explicit use of `v1` suffixes. None of the contracts are upgradeable.

Dictionaries are deployed using solady's 
[SSTORE2](https://github.com/Vectorized/solady/blob/main/src/utils/SSTORE2.sol) contract for gas 
optimization. In theory, this repo can be forked to include any other dictionary. Later versions 
will allow adding new dictionaries onchain without forking.

```
src/
|---- IkhaaliNamesV1.sol # Main Interface + enum definitions
|---- khaaliNamesV1.sol # Main contract
|---- dictionary/
      |---- IkhaaliDictionaryV1.sol # Dictionary Interface
      |---- khaaliDictionaryV1.sol # Abstract Base (SSTORE2 + packing/unpacking)
      |---- ColorDictionary.sol # 50 colors
      |---- AnimalDictionary.sol # 350 animals
      |---- AdjectiveDictionary.sol # 1200 adjectives
|---- account/ # todo example use  #
```

### Dictionary storage

Word data is stored on-chain using 
[SSTORE2](https://github.com/Vectorized/solady/blob/main/src/utils/SSTORE2.sol) from solady. Each 
word is packed into `bytes16` (max 16 characters per word) and deployed as contract bytecode, which
is significantly cheaper than using storage slots:

- Write: ~200 gas/byte vs ~22,100 gas per storage slot
- Read: `EXTCODECOPY` (~2,600 gas) vs `SLOAD` (~2,100 gas)
- Max words per data contract: ~1,500 (within the 24KB bytecode limit)

> [!TIP]
> See actual dictionary data ~ [`colors`](./data/colors.txt), [`adjectives`](./data/adjectives.txt), [`animals`](./data/animals.txt)
